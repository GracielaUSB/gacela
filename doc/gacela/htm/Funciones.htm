Funciones#

subP fun nombre
: (arg1,...,argn) -> tipo
{pre E1} {post E2} {bound T}
 |[ S' >> E ]| 		tipo es el tipo del resultado de la función
nombre es un identificador que representa el nombre de la función
arg1,...,argn son los parámetros formales
E1 es una expresión que representa la precondición
E2 es una expresión que representa la postcondición
T es una expresión que representa la cota
S' es una secuencia de acciones
E expresión que representa el resultado de la función

Cuando la secuencia de acciones es vacía, se puede usar también la siguiente construcción

subP fun nombre
: (arg1,...,argn) -> tipo
{pre E1} {post E2} {bound T}
= E 		tipo es el tipo del resultado de la función
nombre es un identificador que representa el nombre de la función
arg1,...,argn son los parámetros formales
E1 es una expresión que representa la precondición
E2 es una expresión que representa la postcondición
T es una expresión que representa la cota
E es una expresión que representa el resultado de la función

Los parámetros de una función son todos de entrada y dentro de la función pueden ser utilizados como constantes.

Dentro de las funciones no está permitido utilizar la instrucción random, ni operaciones de entrada y salida.

En la declaración de funciones, la precondición y la postcondición son opcionales.
La cota es obligatoria únicamente cuando las funciones son recursivas.

Corrección de funciones#

Sea f una función cuya definición es de la forma siguiente: 

fun f :(x1:t1,...,xn:tn) -> t {pre Pre_f} {post Post_f} |[ A >> e]|

La corrección de la función es la corrección de la tripleta 

 {Pre_f}A{Post_f(f:=e)}

Cuando el cuerpo de la función es vacío, esta triplesa se reduce a 

 [Pre_f ==> Post_f(f:=e)]

Funciones recursivas#

Cuando se definen grupos de una o más funciones mutuamente recursivas (por ejemplo, cuando la función f llama a la función g en su cuerpo y vice-versa), cada llamada a una función del grupo es considerada como una llamada recursiva. Las funciones recursivas deben tener asociadas una función de cota. Dentro del cuerpo de las funciones, además del cumplimiento de la precondición, es necesario que las llamadas recursivas, hagan disminuir la cota, sin que ésta llegue a ser menor que cero.

La cota es únicamente utilizada para demostrar la terminación de las definiciones de las funciones. Las llamadas a funciones definidas recursivamente, ubicadas fuera de las definiciones de éstas no necesitan condición alguna sobre la cota para estar definidas.

Nota
Faltaría explicar como determinar si una llamada es recursiva o no.

Corrección de llamadas no recursivas de funciones.#

Sean f una función con n argumentos x1,...,xn, precondición Pre_f y postcondición Post_f, A una acción que contiene la expresión f(e1,...,en), Q un predicado, C el nombre de una constante no utilizada en el programa y B una acción tal que al remplazar en B las ocurrencias de C por f(e1,...,en) se obtiene la acción A. 

[pmdi(A,Q) == 
    def(e1) /\ ... /\ def(en) /\
    Pre_f(x1,...,xn := e1,...,en) /\
    (Post_f(x1,...,xn,f := e1,...,en,C) ==> 
      pmdi(B,Q))].

Ejemplo#

Suponiendo que se tiene la definicón de factorial como sigue: 

 fun fac: (N:int) -> int {pre N>=0} {post fac=(%prod i : 1<=i<=N : i)} 
 ...

La tripleta {n=2} x:= fac(n+1) {x=6} equivale a la fórmula: 

n=2 ==> 
  def(n+1) /\ Pre_fac(N := n+1) /\ 
  (Post_fac(N,fac := n+1,C) ==> (x=6)(x := C))
== {sustitución, simplificación}
n=2 ==>
  n+1>=0 /\ 
  (C=(%prod i : 1<=i<=n+1 : i) ==> C=6)
== {sustituyendo n por 2, simplificación}
n=2 ==> (C=(%prod i : 1<=i<=3 : i) ==> C=6)
== {simplificación, [P ==> P == true], [P ==> true == true]}
true

Corrección de llamadas de funciones recursivas.#

Sean f una función con n argumentos x1,...,xn, precondición Pre_f, postcondición Post_f y cota T_f, A una acción dentro del cuerpo de f

que contiene la expresión f(e1,...,en), Q un predicado, C el nombre de una constante no utilizada en el programa y B una acción tal que al remplazar en B las ocurrencias de C por f(e1,...,en) se obtiene la acción A. 

[pmdi(A,Q) == 
    def(e1) /\ ... /\ def(en) /\
    Pre_f(x1,...,xn := e1,...,en) /\
    0 <= T_f(x1,...,xn := e1,...,en) < T_f /\
    (Post_f(x1,...,xn,f := e1,...,en,C) ==> 
      pmdi(B,Q))].

Discusión#

Por qué dentro de las funciones no está permitido utilizar la instrucción random ni operaciones de entrada y salida? -- Julio

Porque son funciones en el sentido matemático, para una misma entrada tiene que retornar siempre el mismo valor. Si te permitiera hacer random o consultarle al usuario un valor, sería imposible garantizar que fueran funciones de verdad.

Es como cuándo derivas en función de x, si le das un valor a x ya sería una constante y te daría cero y no otra función. jH²Or
