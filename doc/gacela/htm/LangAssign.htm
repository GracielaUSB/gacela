Asignación paralela#

Sintaxis#

Ax1,...,xn := E1,..,Enx1,...,xn variables,
E1,..,En expresiones.

Condiciones de corrección#

Dado predicados P y Q, expresiones E1,..,En y variables x1,...,xn, la tripleta 

 {P} x1,...,xn := E1,..,En {Q}
es equivalente a la fórmula 
 [P ==> def(E1) /\.../\ def(En) /\ Q(x1,...,xn := E1,...,En)]

En otras palabras, cualquiera sea el valor de las variables, cuando la precondición P es verdadera, las expresiones E1,..,En pueden ser calculadas en tiempo finito, y la postcondición es verdadera una vez remplazadas las ocurrencias de las variables xi por las expresiones Ei correspondientes.

Precondición más débil#

Lo mínimo que se le puede pedir a una precondición para que garantice que una asignación termina en un estado que satisface Q es 
[ pmd(x1,...,xn := E1,...,En,Q) 
    ==
  def(E1) /\.../\ def(En) /\ Q(x1,...,xn := E1,...,En) ]
La pmdi de la asignación se define de igual forma. 
[ pmdi(x1,...,xn := E1,...,En,Q) 
    ==
  def(E1) /\.../\ def(En) /\ Q(x1,...,xn := E1,...,En) ]

Traducción a Java#

La traducción de una asignación paralela es de la forma

TT( {P} x1,...,xn := E1,..,En {Q} ) =TT( {P} ) TT( x1,...,xn := E1,..,En ) TT ( {Q} )

  • Ver en las tripletas la traducción de {P} y de {Q} .
  • Una vez demostrada la fórmula asociada, es posible remover la traducción de {Q} .

Dado que el lenguaje java no posee asignación paralela, la traducción se raliza en base a asignaciones sencillas, bloques y declaración de constantes.

Casos particulares#

  • Cuando n=1 (es decir, sólo hay una asignación):
TT( x1 := E1 ) = TT(x1) "=" TT(E1) ";"
  • Cuando n>1 pero no interviene ninguna expresión de índice de arreglo en esta acción,
es necesario en el peor de los casos guardar en constantes temporales el valor de n-1 de los Ei para luego asignar secuencialmente la expresión restante y las n-1 constantes a las variables por asignar. El tipo tyi de la i-ésima constante definida es el de su variable correspondiente:
TT( x1,...,xn := E1,..,En ) ="{" "final" ty2 _tmp2 "=" TT(E2) ";" ... "final" tyn _tmpn "=" TT(En) ";" TT(x1) "=" TT(E1) ";" TT(x2) "=" _tmp2";" ... TT(xn) = _tmpn";" "}"
  • Finalmente, cuando hay más de un caso y algunos índices de arreglo intervienen, es necesario también guardar en temporales los valores de las expresiones de índice.

Simplificaciones. #

Cuando la variable xi a asignar no aparece en las otras expresiones ni en los índices de arreglo, se puede omitir el guardar su valor en una constante antes de asignar.

Ver también las tripletas, las expresiones y los arreglos.
