Arreglos#

Los arreglos en USB-GCL pueden tener una o más dimensiones, según se indica en el tipo. Los índices de las casillas de un arreglo de dimensión n van entre 0 y n-1. Más generalmente, para una matriz de dimensiones n1xn2x...xnm los índices de las casillas son de la forma [x1]...[xn] en donde cada xi es un valor entre 0 y ni-1.

Tipo arreglo#

El tipo de un arreglo tiene la forma array[E1]...[En] of t donde t es el tipo de las casillas del arreglo, n es el número de dimensiones y E1,...,En son expresiones indicativas del tamaño de cada una de esas dimensiones. Las expresiones E1,...,En se pueden omotir, de manera que el tipo de un arreglo también se puede escribir como array[]...[] of t. El tamaño del arreglo se obtiene de su inicialización.

Inicialización de arreglos#

Para inicializar las constantes y las variables de tipo arreglo se pueden usar dos construcciones: En una de ellas se dan los valores de cada entrada del arreglo explícitamente mediante n expresiones; en la otra se dan n variables, n expresiones enteras, y una expresión en base a las variables que permite asignar valores a cada una de las entradas del arreglo.

E [E0,...,En] E0,...,En expresionesInicialización
de un arreglo
E %[ x1,...,xn : E1,...,En : E ]% x1,...,xn variables,
E1,...,En expresiones		 Creación de una matriz de n dimensiones

En la primera forma de inicialización, se provee una expresión para cada casilla del arreglo. En la segunda construcción, el valor de la casilla de índice [y1]...[yn] (donde cada yi es tal que 0<=yi<Ei) es el valor de la expresión e(x1,...,xn := y1,...,yn).

Las condiciones de definición de una inicialización de un arreglo A de n dimensiones son: 

[def([E0,...,En]) == def(E1) /\ ... /\ def(En)]
[def(%[ x1,...,xn : E1,...,En : E ]%) == 
  def(E1) /\ ... /\ def(En) /\
  0<=E1 /\ ... /\ 0<=En /\  (forall x1,...,xn: 0<=x1<E1,...,0<=xn<En: def(E))]

Declaración de variables y constantes de tipo areglo#

La declaración es similar a la de otras variables y constantes. 

 const A = E : array[E1]...[En] of t
;var A := E : array[E1]...[En] of t

Ejemplos#

  1. const A = [[1,2][3,4]] : array[2][2] of int
    Declaración de un arreglo constante A de tamaño2x2
  2. var b := %[i,j : 2,2 : 2*i+j+1]% : array[2][2] of int
    Declaración de un arreglo variable b de tamaño 2x2.
    El valor de b[0][0] es (2*i+j+1)(i,j:=0,0) = 1
    El valor de b[0][1] es (2*i+j+1)(i,j:=0,1) = 2
    El valor de b[1][0] es (2*i+j+1)(i,j:=1,0) = 3
    El valor de b[1][1] es (2*i+j+1)(i,j:=1,1) = 4

Arreglos como expresiones.#

Para utilizar un arreglo de n dimensiones en una expresión, es necesario proveer n expresiones que servirán de índices del arreglo. Dichas expresiones deben tener su valor dentro del rango de la correspondiente dimensión.

Sea A un arreglo de n dimensiones.

La función length(A,i) retorna el tamaño de un arreglo A en la dimensión i donde 0<=i<n.

Ejemplo#

const A = %[i,j,k : 2,3,4 : i+j-k ]% : array[][][] of int
  { length(A,0)=2 /\ length(A,1)=3 /\ length(A,2)=4 }

Las condiciones de definición de una expresión de acceso a los elementos de un arreglo A de n dimensiones son: 

[def(A[E1]...[En]) == 
  def(E1) /\ ... /\ def(En) /\ 
  0<=E1<length(A,0) /\ ... /\ 0<=En<length(A,n-1)]
