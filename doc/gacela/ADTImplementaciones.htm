Implementaciones de Tipos de Datos (CDT)#

La implementación de un tipo de datos abstracto se realiza mediante un tipo de datos concreto en donde se escoge una representación y se implementan las operaciones sobre el tipo de manera de cumplir con la especificación dada. Un tipo de datos concreto puede implementar más de un tipo de datos abstracto; en ese caso, debe implementar todas las operaciones de cada uno de los tipos abstractos.

Sintaxis de implementaciones#

data type ty(tyargs) |[

 const c1,..,cn : type
 var   v1,..,vm : type

{inv I}

implements adt { coupling }

proc p : (...) {pre Pre_p} {post Post_p} {bound Bound_p} |[...]|

fun f : (...) -> type {pre Pre_f} {post Post_f} {bound Bound_b} |[...]|

meth m : (...) {pre Pre_m} {post Post_m} {bound Bound_m} |[...]|

]|

La representación de un tipo de datos concreto consiste en un conjunto de constantes y variables. La representación concreta no tiene que coincidir con la de los tipos abstractos que implementa. 

 const c1,..,cn : type
 var   v1,..,vm : type

Una implementación comporta un invariante de representación, el cual se debe cumplir una vez construido un valor del tipo, así como al terminar la ejecución de cada método. 

{inv I}

Cuando un tipo de datos concreto implementa un tipo de datos abstracto adt, se debe proveer un mecanismo que permita calcular la correspondiente representación abstracta a partir de la representación concreta. Esto se expresa como un conjunto de ecuaciones de la forma componenteDeRepresentaciónAbstracto = ExpresiónSobreLaRepresenaciónConcreta. 

implements adt { coupling }
separadas por coma.

Un tipo de datos concreto puede contener procedimientos y funciones con las mismas características que en los programas. 

proc p : (...) {pre Pre_p} {post Post_p} {bound Bound_p} |[...]|

fun f : (...) -> type {pre Pre_f} {post Post_f} {bound Bound_b} |[...]|

Los tipos de datos concretos pueden contener métodos, que son parecidos a los procedimientos, salvo que pueden adicionalmente ver y modificar los elementos del modelo de representación. Otro punto en el que difieren de los procedimiente es que es posible mencionar los valores iniciales de las variables en la postcondición, por ejemplo en el caso de una variable v de la representación, la postcondición de un método puede hacer referencia a una constante v' que representa el valor inicial de la variable v. 

meth m : (...) {pre Pre_m} {post Post_m} {bound Bound_m} |[...]|

Dentro de las operaciones de un tipo de datos abstracto es posible utilizar la función ty(m1,...,mn) donde ty es el nombre del tipo de datos y m1,...,mn son los componentes de la representación. El resultado de esta función es un nuevo valor del tipo de datos.

Constructores de Tipo#

Hasta el momento se han presentado los mecanismos de consulta y modificación de los valores contenidos en un tipo de dato, sin embargo aún no se tiene forma de crear una variable (o constante) que sea de un tipo de dato. Justamente está es la labor de los contructores, crear una variable (o constante) del tipo de dato.

En toda implementación de tipo de dato existe una función "escondida" que sólo puede ser accedida desde sus funciones, métodos y procedimientos, ésta tiene el mismo nombre que el tipo de dato y que recibe como parámetros todas las variables y constantes del modelo de representación en el orden en que fueron declaradas, por ejemplo si se tiene la siguiente declaración y modelo de representación de un tipo de dato 

data type Tipo |[
    const C1:t1
    const C2:t2
    var v1:t3
    var v2:array[] of t4

Su construtor será 

Tipo (c1,c2,v1,v2)

Donde,

  • c1 es de tipo t1
  • c2 es de tipo t2
  • v1 es de tipo t3
  • v2 es de tipo array of t4

Implementación de diccionarios[1]#

Ejemplo 

data type DicoArreglo(_T0,_T1) |[

    const MAX : int
    var claves : array[] of _T0
    var valores : array[] of _T1
    var tam : int

    {inv 0 <= tam <= MAX /\ (%forall i,j : 0<=i<j<tam : claves[i]!=claves[j]) }

    implements Dico { 
      conoc = (%set i : 0<=i<tam: claves[i]),
      tabla = (%set i : 0<=i<tam: pair(claves[i],valores[i]))
    }                

    fun vacio : (tamMax:int) -> DicoArreglo(_T0,_T1)
    {pre tamMax > 0} 
    {post vacio.MAX = tamMax /\ vacio.tam=0}
    = DicoArreglo(tamMax,%[i : tamMax : witness of _T0]%,
                  %[i : tamMax : witness of _T1]%, 0)

    meth agregar : (in x : _T0, in y :_T1)
    {pre !(%exists i : 0<=i<tam : x=claves[i]) /\ tam<MAX }
    {post %{i : 0<=i<tam : claves[i]}% = %{i : 0<=i<tam' : claves'[i]}% + {x} /\ 
          (%forall i : 0<=i<tam :
               claves[i]=x ==> valores[i]=y /\ 
               (%forall j : 0<=j<tam' : claves[i]=claves'[j] ==> valores[i]=valores'[j]
               )) /\
           tam = tam' + 1
    } 
    |[
        claves[tam] := x
       ;valores[tam] := y
       ;tam := tam + 1
    ]|


    meth eliminar : (in x :_T0)
    {pre (%exists i : 0<=i<tam : x=claves[i])}
    {post %{i : 0<=i<tam : claves[i]}% = %{i : 0<=i<tam' : claves'[i]}% - {x} /\ 
          (%forall i,j : 0<=i<tam, 0<=j<tam' : 
               claves[i]=claves'[j] ==> valores[i]=valores'[j]) /\
          tam = tam' - 1
    }
    |[
        var k := 0 : int
       ;{inv 0<=k<tam /\ (%exists i : k <= i< tam : x=claves[i])}
        {bound tam-k}
        do claves[k]!=x -> k := k + 1 od
       ;claves[k] := claves[tam-1]
       ;valores[k] := valores[tam-1]
       ;tam := tam - 1
    ]|

    fun obtener : (d:Dico, x:_T0) -> _T1
    {pre (%exists i : 0<=i<d.tam : x=d.claves[i]) }
    {post obtener = (%first i: 0<=i<d.tam, d.claves[i]=x : d.valores[i]) } 
    |[
        var k := 0 : int
       ;{inv 0<=k<d.tam /\ (%exists i : k <= i< d.tam : x=d.claves[i])}
        {bound d.tam-k}
        do d.claves[k]!=x -> k := k + 1 od
       >> d.valores[k]
    ]|
 
    fun member : (d:Dico, x:_T0) -> boolean
    {pre true }
    {post member = (%exists i : 0<=i<d.tam : x=d.claves[i])}
    |[
        var k := 0 : int
       ;{inv 0<=k<=d.tam /\ 
             (%exists i : 0 <= i< d.tam : x=d.claves[i]) ==
             (%exists i : k <= i< d.tam : x=d.claves[i])
        }
        {bound d.tam-k}
        do k!=d.tam /\ d.claves[k]!=x -> k := k + 1 od
       >> k!=d.tam
    ]|

]|
