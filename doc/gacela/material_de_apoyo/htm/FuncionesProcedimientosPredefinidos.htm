Procedimientos y Funciones Predefinidas#

Lectura y Escritura de Archivos#

 fun abrirArchivoLeyendolo(nombreDeArchivo:String) -> String 
{ pre nombreDeArchivo es el path de un archivo regular existente para el cual
      se tiene permiso de lectura }
{ post abrirArchivoLeyendolo es el identificador del archivo abierto para lectura }

Recibe el 'path' o dirección del archivo que se quiere abrir para lectura y devuelve el identificador de ese archivo una vez abierto. 

 fun abrirArchivoCreandolo(nombreDeArchivo:String) -> String
{ pre nombreDeArchivo es el path de un archivo regular existente para el cual
      se tiene permiso de escritura o no existe pero puede ser creado }
{ post abrirArchivoLeyendolo es el identificador del archivo abierto para escritura }

Recibe el 'path' o dirección del archivo que se quiere abrir para escribir en él. Esta escritura se hará desde el inicio del archivo sobreescribiendo el contenido del mismo, si existía. Devuelve el identificador del archivo abierto. 

 fun abrirArchivoAnexandole(nombreDeArchivo:String) -> String
{ pre nombreDeArchivo es el path de un archivo regular existente para el cual
      se tiene permiso de escritura o no existe pero puede ser creado }
{ post abrirArchivoLeyendolo es el identificador del archivo abierto para escritura }

Recibe el 'path' o dirección del archivo que se quiere abrir para escribir en él. Si el archivo existía, esta escritura se hará a continuación de los datos que éste contenía. Devuelve el identificador del archivo abierto. 

 fun leerLineaDeArchivo:(id:String) -> String
{ pre id es el identificador de un archivo }
{ post leerLineaDeArchivo contiene la próxima linea no leída del archivo }

Recibe un identificador que ha sido obtenido a través de una de las tres funciones anteriores y devuelve la siguiente linea a leer del archivo, si no quedan lineas por leer se devuelve el String vacio (""). 

 proc escribirLineaEnArchivo:(in id:String, in linea:String)
{ pre id es el identificador de un archivo }
{ post se ha escrito el String linea en una nueva linea del archivo indicado por el id }

Recibe el identificador de un archivo (identificador que ha sido obtenido a través de alguna de las tres primeras funciones de esta sección) y un String, y escribe en una nueva linea de ese archivo el String dado como parámetro. 

 proc cerrarArchivo:(in id:String) 
{ pre id es el identificador de un archivo }
{ post se ha cerrado el archivo asociado a id }
Recibe el identificador de un archivo (identificador que ha sido obtenido a través de alguna de las tres primeras funciones de esta sección). Luego de invocar a este procedimiento no se podrá usar id como argumento de leerLineaDeArchivo ni escribirLineaEnArchivo. Se invoca luego de realizar las operaciones de lectura o escritura relacionadas al archivo asociado a id.

Disponibles a partir de GCL versión 2.3.493
