Lazo Do#

Sintaxis#

A {inv I} {bound T}
do B1 -> S1
| ...
| Bn -> Sn
od		I es una expresión que representa el invariante del lazo,
T es una expresión que representa la cota,
B1,...,Bn son las expresiones de guardia, y
S1,..,Sn son sus respectivas secuencias de acciones.

Para comprender el comportamiento del lazo, se puede imaginar que éste representa el valor de la variable DO en la ecuación recursiva: 
DO : [DO == if (!B1 /\ ... /\ !Bn) -> skip | B1 -> S1 ; DO | ...  | Bn -> Sn ; DO fi]

Condiciones de correción#

Invariante#

El invariante es una condición que debe cumplirse en cada paso de una iteración. Fue introducida por R. Floyd en 1967.
EjFloyd.jpg

La construcción básica del lazo 

 {P} {inv I} {bound T}
 do   B1 -> S1 | ... | Bn -> Sn 
 od {Q}

equivale a:

  1. Inicialización garantiza invariante
    P ==> I
  2. Invariante garantiza guardias calculables
    I ==> def(B1)/\.../\def(Bn)
  3. Invariante se mantiene
    {I /\ Bi} Si {I} (para i entre 1 y n)
  4. Lazo Termina
    {I /\ Bi /\ C=T} Si {C>T>=0}
  5. Salida garantiza postcondición
    (I /\ !B1 /\ ... /\ !Bn) ==> Q

Precondición más débil#

Siguiendo la interpretación del lazo en base al condicional, se puede deducir un método para calcular la precondición más débil del lazo. En este caso se hará para un lazo con un único comando con guardia:

Sea DO el lazo do B -> S od 

pmd(Do,Q)
== { visto el lazo como condicional }
pmd(if !B -> skip | B -> S ; DO fi,Q)
== { pmd del condicional }
def(B) /\ (!B \/ B) /\ (!B ==> pmd(skip,Q)) /\ (B ==> pmd(S ; DO,Q))
== { [!B \/ B == true], [true /\ A == A], pmd de skip y secuencia }
def(B) /\ (!B ==> Q) /\ (B ==> pmd(S,pmd(DO,Q)))
== { [A ==> B == !A \/ B], [!!A == A] }
def(B) /\ (B \/ Q) /\ (!B \/ pmd(S,pmd(Do,Q))

De esta deducción sacamos que para calcular la precondición más débil debemos resolver la ecuación (obtenida de la deducción anterior, remplazando pmd(DO,Q) por X): 

 X : [X == def(B) /\ (B \/ Q) /\ (!B \/ pmd(S,X))]
Si bien es posible calcular soluciones para esta ecuación por aproximaciones sucesivas, en GCL optamos por dar mayores responsabilidades al programador exigiendole que él proponga obligatoriamente un invariante y una cota para cada lazo que escriba.

Definimos a partir de la precondición más débil una función que llamamos Precondición más débil por invariante, abreviado pmdi que, en el caso del lazo se define como: 

                [ I ==> (%forall i : 0<=i<n : def(Ei)) ], 
    [ (%forall i : 0<=i<n : (I /\ Bi /\ C=T) => pmdi(Si,I/\ 0<=T<C)) ],
                [I /\ (%forall i : 0<=i<n : !Bi) ==> Q]
   - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
    [ pmdi({inv I}{bound t}do B1 -> S1 | ... | Bn -> Sn od,Q) == I ]
(donde C es el nombre de una constante que no aparece en el programa.)

Esta regla se lee: Una vez demostrado que el invariante garantiza que los guardias se pueden calcular, que el invariante se mantiene decreciendo la cota y que la post-condición es asegurada por el invariante y la negación de los guardias, la pmdi del lazo es el invariante.

Traducción a Java#

La traducción del lazo es de la forma

TT( {P} {inv I} {bound T} do B1 -> S1 |...| Bn -> Sn od {Q} =TT( {P} ) TT( {I} ) TT( {inv I} {bound T} do B1 -> S1 |...| Bn -> Sn od ) TT ( {Q} )

Cuando hay un sólo caso, la traducción del lazo se raliza como un lazo while de Java.

TT( {inv I} {bound T} do B -> S od )=while (TT(B)) {final int C = T; TT{S); TT( { I /\ 0<=T<C } )}

Cuando hay más de un caso, la traducción del lazo se raliza como un lazo while con etiqueta y salidas con la construcción break.

TT( {inv I} {bound T} do B1 -> S1 |...| Bn -> Sn od )= lab:while (true) { final int C = T; if (TT (B1)) {TT{S1)} ... else if (TT (Bn)) {TT{Sn)} else break lab; TT( { I /\ 0<=T<C } }

  • Ver en las tripletas la traducción de {P} y de {Q} .
  • Una vez demostrada la fórmula asociada, es posible remover la traducción de {Q} .
